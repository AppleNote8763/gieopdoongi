# 기업뚱이 작업 히스토리

작성일: 2026-05-22

이 문서는 최근 며칠 동안 `기업뚱이` 프로젝트에서 어떤 흐름으로 기능을 만들고 수정했는지 정리한 기록이다. 코드 구현 방식보다는 서비스 기능과 화면 변화 중심으로 작성했다.

---

## 1. 초기 앱 구조와 기본 로드맵 생성

- AI 취업 준비 플래너의 기본 흐름을 만들었다.
- 사용자가 목표 기업, 희망 직무, 보유 역량, 경력 수준을 입력하면 AI가 취업 준비 로드맵을 생성하도록 구성했다.
- 화면은 크게 3단계로 나뉘었다.
  - 목표 설정
  - 로드맵 생성
  - 진행 관리
- Node.js + Express 서버가 `public` 폴더의 프론트엔드 파일을 제공하는 구조로 시작했다.
- 프론트엔드는 HTML, CSS, Vanilla JavaScript 기반으로 구성했다.
- Gemini API를 연결해 로드맵 생성 기능을 붙였다.

## 2. Supabase 인증과 사용자별 저장 기능

- Supabase Authentication을 연결했다.
- 이메일 기반 회원가입, 로그인, 로그아웃 기능을 추가했다.
- 로그인 상태가 새로고침 후에도 유지되도록 세션 복원 흐름을 만들었다.
- 로그인한 사용자만 로드맵을 저장하고 다시 불러올 수 있게 했다.
- Supabase Database에 `roadmaps` 테이블을 만들고 사용자별 로드맵을 저장하도록 했다.
- 비로그인 사용자도 로드맵 생성은 가능하지만, 저장과 진행률 관리는 로그인 후 가능하도록 구분했다.

## 3. 직무 추천과 역량 추천 기능 추가

- 목표 기업을 입력한 뒤 `직무 찾기` 버튼으로 추천 직무를 받을 수 있게 했다.
- Gemini가 기업과 업종을 보고 직무를 추천하도록 했다.
- Gemini 응답이 불안정하거나 과부하일 때도 앱이 멈추지 않도록 fallback 추천 목록을 넣었다.
- 현대자동차, 기아, 모빌리티, 게임, 커머스, 물류, 오디오 등 일부 산업군은 더 알맞은 직무가 먼저 나오도록 보정했다.
- 희망 직무는 드롭다운 추천과 직접 입력을 함께 지원하도록 했다.
- `역량 찾기` 버튼을 추가해 목표 기업과 직무에 맞는 역량을 추천받을 수 있게 했다.
- 추천 역량은 체크박스로 선택할 수 있고, 목록에 없는 역량은 직접 입력할 수 있게 했다.
- 보유 역량을 비워도 초보자 기준 로드맵이 생성되도록 바꿨다.

## 4. 준비 기간과 자격증 입력 기능 확장

- 취업 준비 기간을 선택하는 기능을 추가했다.
- 1개월, 3개월, 6개월, 1년 같은 기본 기간을 선택할 수 있게 했다.
- 직접 텍스트로 기간을 입력할 수 있게 했다.
- 달력에서 시작 날짜와 목표 날짜를 선택할 수 있게 했다.
- 목표 자격증 입력칸을 추가했다.
- 직무별로 관련 자격증을 추천하는 버튼을 추가했다.
- 자격증이 비어 있으면 AI가 직무와 로드맵에 맞는 필수 자격증을 자동으로 판단하도록 했다.

## 5. 로드맵 생성 결과 고도화

- AI 로드맵 결과에 핵심 역량, 보완할 역량, 우선순위, 포트폴리오 방향, 면접 준비, 주차별 실행 계획, 체크리스트가 포함되도록 했다.
- 준비 기간에 따라 체크리스트 일정이 자동으로 나뉘도록 했다.
- 체크리스트 기간이 전부 같은 값으로 반복되지 않도록 전체 기간과 항목 수를 기준으로 분산했다.
- 항목 난이도에 따라 예상 기간이 조금씩 다르게 나오도록 조정했다.
- 기존에 생성된 로드맵을 다시 생성하거나 저장할 수 있는 흐름을 만들었다.
- 로드맵 생성 후 진행 관리 화면으로 이동할 때 미저장 로드맵은 자동 저장되도록 했다.

## 6. 고용24 채용공고 연동

- 고용24 채용공고 OpenAPI 연동 구조를 추가했다.
- 로드맵 생성 시 실제 채용공고를 조회하고, 공고에서 요구하는 역량을 로드맵에 반영하도록 했다.
- 채용공고 결과는 최대 8개까지 분석하도록 했다.
- 같은 조건으로 반복 조회할 때 과도한 호출을 줄이기 위해 캐시를 적용했다.
- 고용24 API 키가 없거나 결과가 없을 때도 앱이 멈추지 않고 안내 문구가 보이도록 했다.
- 진행 관리 화면에는 최신 채용공고 위젯을 추가했다.
- 채용공고 마감일까지 남은 일수를 D-Day 형태로 보여주도록 했다.

## 7. 자격증 시험 일정 반영

- 서버에 주요 자격증 시험 일정 데이터를 넣었다.
- 정보처리기사, SQLD, ADsP, 컴퓨터활용능력 1급 같은 자격증 일정이 로드맵과 매칭되면 함께 보여주도록 했다.
- 진행 관리 화면에 자격증 시험 일정 위젯을 추가했다.
- 직무와 자격증이 연결되는 경우 준비 기간과 체크리스트에 자격증 준비 흐름이 반영되도록 했다.

## 8. 진행 관리 대시보드 고도화

- 진행 관리 화면을 대시보드 형태로 개편했다.
- 목표 직무와 준비 기간 요약을 상단에 보여주도록 했다.
- 목표일까지 남은 일수를 보여주는 D-Day 배너를 추가했다.
- 체크리스트 완료율 기반의 학습 진행률을 만들었다.
- 시작일과 목표일 기준의 시간 진행률을 함께 보여주도록 했다.
- 일정 대비 학습이 뒤처지면 경고 색상이 보이도록 했다.
- 저장된 로드맵을 열면 진행 관리 화면으로 바로 이동하도록 했다.

## 9. 체크리스트 관리 기능 개선

- 체크리스트 항목을 체크하면 완료 상태로 바뀌도록 했다.
- 체크박스뿐 아니라 항목 카드 전체를 눌러도 완료/해제가 되도록 했다.
- 체크리스트 항목에 드래그 핸들을 추가해 우선순위를 바꿀 수 있게 했다.
- 우선순위 변경 후 저장된 로드맵이면 변경된 순서도 서버에 저장되도록 했다.
- 체크리스트 예상 기간을 인라인으로 직접 수정할 수 있게 했다.
- 사용자가 직접 수정한 기간은 이후 자동 재분산 과정에서 덮어쓰지 않도록 했다.
- 나만의 체크리스트 항목을 추가할 수 있게 했다.

## 10. 로드맵 다시 최적화 기능

- 진행 관리 화면에서 로드맵을 다시 최적화하는 기능을 추가했다.
- 새로운 준비 기간을 선택한 뒤 로드맵을 재생성할 수 있게 했다.
- 1개월, 3개월, 6개월, 1년, 직접 입력, 달력 선택을 지원했다.
- 기존에 완료한 체크리스트 항목은 삭제하지 않고 새 체크리스트 앞에 보존하도록 했다.
- 저장된 로드맵을 다시 최적화하면 재생성된 결과도 서버에 저장되도록 했다.
- 달력 선택 영역이 보이지 않거나 숨김 상태가 맞지 않는 문제를 수정했다.

## 11. UI와 반응형 디자인 개선

- 앱 표시명을 `기업뚱이`로 변경했다.
- 헤더의 로그인/회원가입 버튼 크기를 정렬했다.
- 모바일 화면에서 상단 헤더가 공간을 너무 차지하지 않도록 조정했다.
- 로드맵 카드, 진행률 카드, 체크리스트 카드, 배지 스타일을 다듬었다.
- 긴 체크리스트 제목이 카드 밖으로 넘치거나 레이아웃을 깨는 문제를 수정했다.
- 잘린 체크리스트 제목이 있으면 설명 내용을 이용해 전체 제목을 복원하도록 했다.
- 진행 관리 페이지의 카드와 배너, 체크리스트 영역을 모바일에서 보기 좋게 보정했다.

## 12. 주요 오류와 안정화 작업

- 직무 찾기 요청이 JSON 대신 HTML을 받아 오류가 나던 문제를 수정했다.
- Gemini 429/503 과부하 상황에서 fallback 결과를 반환하도록 했다.
- 직무와 역량 추천 요청에 timeout 처리를 추가했다.
- `selectedSkills is not defined` 오류를 수정했다.
- 목표 설정 검증 중 준비 기간 에러 요소가 없어 제출이 막히던 문제를 수정했다.
- 기존 서버가 살아 있어 최신 API가 아닌 fallback HTML을 반환하던 상황을 확인하고 안전 처리를 추가했다.
- 로그아웃 시 localStorage와 sessionStorage를 포함해 상태가 초기화되도록 했다.

## 13. Vercel 배포와 운영 정리

- GitHub 저장소와 Vercel 자동 배포 흐름을 맞췄다.
- `main` 브랜치 기준 자동 배포를 전제로 정리했다.
- Vercel 환경 변수로 Gemini, Supabase, 고용24 키를 등록해야 하는 구조를 문서화했다.
- Supabase Auth의 Site URL과 Redirect URL에 배포 URL을 등록해야 하는 점을 정리했다.
- `HANDOFF.md`와 `README.md`를 갱신해 다음 작업자가 이어받기 쉽도록 했다.

## 14. 오늘 추가 반영한 내용

- 목표 설정 페이지의 `이전 데이터` 카드에서 저장된 로드맵을 삭제할 수 있게 했다.
- 서버에 로그인 사용자 본인 소유 로드맵만 삭제하는 API를 추가했다.
- 삭제 버튼과 로드맵 열기 영역을 분리해 실수로 열기/삭제가 섞이지 않도록 했다.
- 삭제 버튼 색상을 전체 디자인 톤에 맞게 중립적인 회색 계열로 조정했다.
- 진행 관리 페이지의 로드맵 다시 최적화 폼에서 달력 선택 시 `기간 선택` 영역이 밀리거나 겹쳐 보이는 문제를 줄였다.
- 학습 체크리스트를 체크할 때 Supabase 저장을 기다린 뒤 화면이 바뀌던 흐름을 개선했다.
- 체크 즉시 UI와 진행률이 먼저 반영되고, 저장은 뒤에서 짧게 묶어서 처리되도록 했다.
- 빠르게 여러 항목을 눌러도 마지막 상태가 서버에 남도록 저장 큐를 추가했다.

## 15. 현재 남아 있는 후속 후보

- 배포 URL에서 회원가입, 로그인, 로드맵 생성, 저장, 불러오기 전체 흐름을 실제로 다시 테스트하기
- Supabase Auth URL Configuration에 최종 배포 URL이 정확히 들어갔는지 확인하기
- 모바일에서 체크리스트 드래그와 역량 체크박스 사용성을 추가 점검하기
- 자격증 시험 일정 데이터를 서버 하드코딩이 아니라 자동 갱신 구조로 개선하기
- 고용24 API 키가 없거나 결과가 없을 때의 안내 문구를 더 자연스럽게 다듬기
- 현재 문서 변경과 오늘 수정사항을 커밋/푸시할지 결정하기
